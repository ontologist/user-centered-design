## About This Project

This user-centered design requirements gathering website was created to support the HR AI Assistant project. It provides a comprehensive framework for conducting user research, developing personas, and documenting requirements using industry-standard design thinking methodology.

### Technologies Used
- HTML5, CSS3, JavaScript
- jsPDF library for PDF generation
- GitHub Pages for hosting
- Responsive design principles
- Print-optimized CSS

### Methodologies Applied
- Stanford d.school Design Thinking
- IDEO Human-Centered Design
- Nielsen Norman Group UX Research
- GOV.UK Service Design Standards

## Project Purpose

To ensure the HR AI Assistant is built with user needs at the center, preventing common pitfalls of:
- Unclear or shifting requirements
- Stakeholder misalignment
- Feature creep and scope management issues
- Building solutions without user validation
